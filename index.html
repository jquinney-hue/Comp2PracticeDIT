Characteristics of Data & Information

Use Excel to explore the difference between raw data and meaningful information by entering, organising, and processing data.

📋 Part 1 — Enter Raw Data

1

Open a new workbook

Launch Excel. Click File → New → Blank Workbook. Save it immediately as A1_Data_Information.xlsx using Ctrl + S.

2

Label your columns

In Row 1, type these headers: A1: Date, B1: Visitors, C1: Parking Spaces, D1: Total Sales (£), E1: Chemist Sales (£). Bold them with Ctrl + B.

3

Enter the shopping centre data

In Row 2, enter: 18/06/2020 | 2000 | 200 | 150000 | 2000. This is raw data — numbers with no meaning yet.

4

Format the date column

Click cell A2. Press Ctrl + 1 to open Format Cells. Choose Date and pick the format DD/MM/YYYY. Click OK.

📊 Part 2 — Turn Data into Information

5

Calculate parking usage

In F1 type Parking Usage %. In F2 type the formula: =B2/C2*100. This tells us what percentage of parking spaces were needed — now the data has meaning!

6

Calculate average spend per visitor

In G1 type Avg Spend (£). In G2 type: =D2/B2. Round to 2 decimal places: =ROUND(D2/B2,2).

7

Add a context column

In H1 type Context Note. In H2 type: "Sale on popular items in chemist". Notice how context makes data more informative.

8

Add a second row (previous year)

In Row 3, enter the same date one year earlier (18/06/2019) and estimated lower sales values. In I1 type YoY Change (£). In I3 type: =D2-D3 — this shows year-on-year growth.

💡 Key Concept: The numbers in columns A–E are data. Once you add formulas (F–I) that give context and meaning, it becomes information that can inform decisions.

✅ Your Task

  • Add 5 more rows for different days at the shopping centre (make up realistic figures).
  • Use =AVERAGE(B2:B7) to find average daily visitors.
  • In a text box (Insert → Text Box), write 2 sentences explaining what information you can now extract.

8

Insert sparklines

Click cell G2 (next to row 1 of weekly data). Go to Insert → Sparklines → Line. In Data Range, select the 5 weeks of data for that shop. Click OK. Copy the sparkline down to the remaining rows.

9

Highlight sparkline markers

With sparklines selected, go to Sparkline tab → Show → High Point (ticked) and Low Point (ticked). This makes trends instantly visible.

Ensuring Data is Suitable for Processing

Use Excel's Data Validation tool to apply range checks, type checks, presence checks and length checks to a registration form.

🏃 Part 1 — Build a Sports Registration Form

1

Create a new sheet

Add a sheet named A3_Validation. In Column A, list these labels: First Name, Last Name, Age, Shoe Size, Phone Number, Postcode, Date of Birth, Sport.

2

Label Column B as input fields

In B1 type Your Entry. Widen columns A and B so labels are fully visible. Freeze the header row: View → Freeze Panes → Freeze Top Row.

🔢 Part 2 — Apply Validation Rules

3

Range check — Age

Click B3 (next to Age). Go to Data → Data Validation → Data Validation. Set Allow: Whole Number, Data: Between, Min: 10, Max: 80. Click the Error Alert tab: Style = Stop, Title = Invalid Age, Message = Age must be between 10 and 80.

4

Range check — Shoe Size

Click B4. Data Validation → Whole Number → Between, Min: 1, Max: 15. Add an error message: Shoe size must be between 1 and 15.

5

Type check — Date of Birth

Click B7. Data Validation → Allow: Date, Data: between, set a realistic range (e.g. 01/01/1940 to today). Error message: Please enter a valid date in DD/MM/YYYY format.

6

List check — Sport

Click B8. Data Validation → Allow: List. In Source, type: Football,Basketball,Swimming,Tennis,Athletics. This creates a dropdown. The user can only select from your list — preventing typos.

7

Length check — Phone Number (using a formula)

Click B5. Data Validation → Allow: Custom. Formula: =LEN(B5)=11. Error: Phone number must be exactly 11 digits. This is a length check — LEN() counts the number of characters.

8

Presence check — First & Last Name

Click B1. Data Validation → Allow: Custom. Formula: =LEN(B1)>0. Error: This field cannot be left blank. Repeat for B2. This checks the field isn't empty.

📋 Part 3 — Add Input Messages (User Guidance)

9

Add helpful tooltips

For each validated cell, go back into Data Validation → Input Message tab. Add a helpful title and message. E.g. for Age: Title = Enter Age, Message = Please enter your age as a whole number between 10 and 80.

10

Test your validation

Try entering incorrect data in each field (e.g. type 5 for age, or abc for phone). Confirm the error messages appear correctly. Then fill in valid data to check it works.

💡 Validation vs Verification: Validation is an automatic check the computer performs (like the rules above). Verification is a human check — e.g. proofreading or double-entry (typing the same data twice to confirm it matches).

📊 Part 2 — Analyse Your Survey Results

3

Add a COUNTIF for Primary vs Secondary

Below the table, create a summary. In a new cell type: =COUNTIF(B2:B7,"Primary") labelled "Primary Methods" and =COUNTIF(B2:B7,"Secondary") labelled "Secondary Methods".

4

Calculate average scores

Use =AVERAGE(F2:F7) to find average reliability and =AVERAGE(G2:G7) for average accuracy across all methods.

5

Find the most reliable method

Use =MAX(F2:F7) to find the highest reliability score. Then use =INDEX(A2:A7,MATCH(MAX(F2:F7),F2:F7,0)) to return the method name. This is powerful data analysis!

6

Use Conditional Formatting to highlight

Select the Reliability column (F2:F7). Go to Home → Conditional Formatting → Color Scales → Green-Yellow-Red. High scores go green, low scores go red — instant visual analysis.

🔍 Part 3 — Simulate Survey Data

7

Create a new sheet: Survey_Results

Simulate 15 classmates' survey responses about post-16 choices. Columns: Respondent, Age, Gender, Choice (Stay in School / College / Apprenticeship / Job), Reason.

8

Count responses for each choice

Use =COUNTIF(D2:D16,"College") (and similar) to count how many chose each option. Create a bar chart from these counts.

9

Filter by gender

Select your data → Data → Filter. Use the Gender dropdown to view only male or female responses. Notice how filtering small samples can give very different pictures of the data.

💡 Data Collection Features to consider: Sample size, who is in the sample, when data was collected (currency), where it was collected, and what method was used — all affect how reliable and useful the data is.

Quality of Information & Decision Making

Use Excel to explore how inaccurate or biased data leads to poor decisions — using the exercise scenario.

⚠️ Part 1 — Simulate Biased Survey Data

1

Create sheet: A5_QualityData

Set up a table for the exercise survey about exercise. Columns: Respondent ID, Age, Current Exercise (hrs/week), Opinion on Exercise (Positive/Negative/Neutral), Health Condition (Yes/No).

2

Enter a biased sample (10 people)

Make all 10 respondents aged 65–80 with health conditions and low exercise levels. Make most opinions "Negative". This represents a biased sample.

3

Analyse the biased data

Use =COUNTIF(D2:D11,"Negative") to count negative opinions. Use =AVERAGE(C2:C11) for average exercise hours. Create a pie chart showing opinion distribution.

4

Enter a balanced sample (10 more people)

In rows 12–21, add 10 people aged 16–45, mixed health, higher exercise levels, mostly Positive opinions.

5

Compare the two charts

Create a second pie chart for the balanced group. Place both charts side by side. The difference will clearly show how who was surveyed affects the results dramatically.

🏛️ Part 2 — Decision Impact Matrix

6

Create a decision impact table

Create a new table with columns: Decision Maker, Decision Based on Inaccurate Data, Impact, Severity (1-5). Fill in rows for: Local Council, Sports Clubs, Schools, GPs, Gym Owners.

7

Apply conditional formatting to Severity

Select the Severity column. Home → Conditional Formatting → Icon Sets → 3 Traffic Lights. High severity = red, medium = amber, low = green.

8

Sort by severity

Click any cell in your table. Go to Data → Sort. Sort by Severity column, Largest to Smallest. This lets you quickly see which decisions would be most harmful.

💡 GIGO — Garbage In, Garbage Out: If data entered into a system is inaccurate, the information produced will also be inaccurate. No matter how well Excel processes data, if the input is wrong, the output will be wrong too.

Sectors That Use Data Modelling

Build a real data model in Excel — a smartphone comparison tool — to understand how organisations use data modelling for decisions.

📱 Part 1 — Smartphone Comparison Model

1

Create sheet: A6_DataModel

Set up columns: Phone Model, Price (£), Battery (hrs), Storage (GB), Camera (MP), Rating (1-5), Weighted Score.

2

Enter 5 phone models

Add realistic data for 5 different smartphones. Use manufacturer websites or comparison sites to get accurate figures. Entering real data makes the model genuinely useful.

3

Set weighting factors

In a separate area, create a "Weights" section. E.g: Price weight: 0.3, Battery: 0.2, Storage: 0.2, Camera: 0.15, Rating: 0.15. Weights must add up to 1.0 — check with =SUM(weights).

4

Normalise scores (0–100 scale)

For each metric, calculate a normalised score. For price (lower is better): =MIN(B2:B6)/B2*100. For battery (higher is better): =C2/MAX(C2:C6)*100. Do this for all metrics in helper columns.

5

Calculate a Weighted Score

In the Weighted Score column, multiply each normalised score by its weight and sum them. E.g.: =(NormPrice*0.3)+(NormBattery*0.2)+(NormStorage*0.2)+(NormCamera*0.15)+(NormRating*0.15). This gives each phone a single comparable score.

6

Highlight the winner

Select the Weighted Score column. Conditional Formatting → Top/Bottom Rules → Top 1 Item. Format it with a green fill to highlight the best overall phone based on your model.

🏦 Part 2 — Bank Branch Closure Model

7

Create a second table on the same sheet

Columns: Branch Location, Monthly Customers, Annual Cost (£), Online Banking Users %, Nearest Alternative Branch (miles), Elderly Population %.

8

Calculate cost per customer

Add a column: =C2/B2/12 — this gives the monthly cost to serve each customer. Branches with very high cost-per-customer are candidates for closure.

9

Add an IF formula for closure recommendation

In a new column called Recommend Closure?, use: =IF(AND(D2>70,E2<5,B2<500),"Yes","No"). This says: if most customers bank online AND there's a nearby branch AND footfall is low → recommend closure.

10

Test your model with What-If Analysis

Change the threshold values in your IF formula. What happens if you lower the online banking threshold from 70% to 50%? How does this change the recommendations? This is what-if modelling.

💡 Why organisations use data models: They allow decisions to be tested before real-world action, multiple factors to be weighed simultaneously, and outcomes to be compared quickly — saving time, money, and reducing risk.

  • Change the weights in your phone model so that price is now 0.5 (most important). How does the winning phone change?
  • In a text box, explain why two different people (a student vs a professional) might want different weightings.

Threats to Individuals

Use Excel to catalogue data threats, track potential misuse scenarios, and produce a structured analysis to support your infographic.

🔐 Part 1 — Personal Data Audit

1

Create sheet: A7_Threats

In this sheet, list all the types of data collected about you. Columns: Data Type, Who Collects It, Where It's Stored, Is It Sensitive? (Y/N), Potential Threat, Risk Level (1-5).

2

Fill in at least 10 data types

Examples: Name and address (school), browsing history (Google), purchase history (Amazon), location data (mobile phone provider), photos (social media), medical records (NHS). Be specific about where data is stored.

3

Assign risk levels and apply colour coding

Rate each data type's risk from 1 (low) to 5 (high). Apply Conditional Formatting → Color Scales to the Risk Level column. This gives you a quick visual threat assessment.

4

Count sensitive data entries

Below the table use: =COUNTIF(D2:D11,"Y") labelled "Sensitive Data Items". Also calculate: =AVERAGE(F2:F11) for average risk level. These numbers can go in your infographic.

⚡ Part 2 — Threat Scenarios

5

Create a Threats table

New table, columns: Threat Type, Description, Target, Prevention Method, How Common? (1-5). Add rows for: Identity Theft, Phishing, Inaccurate Data Storage, Targeted Advertising, Data Breach, Invasion of Privacy.

6

Categorise targets using dropdown validation

In the Target column, apply Data Validation → List with: All Individuals,Vulnerable People,Elderly,Children,Businesses,Everyone. This keeps your analysis consistent.

7

Filter for high-risk threats

Apply a Filter to your table. Filter "How Common?" to show only values of 4 or 5. These are the threats most worth focusing on for your infographic.

🛡️ Part 3 — Prevention Checklist

8

Build a precautions table

Create a third table: Precaution, Protects Against, Difficulty (Easy/Medium/Hard), Cost (Free/Paid), Completed? (checkbox).

9

Add checkboxes to the Completed column

Click E2. Go to Developer tab → Insert → Form Controls → Check Box. (If Developer tab isn't visible: File → Options → Customise Ribbon → tick Developer.) Place checkboxes in each row.

10

Count completed precautions

Link each checkbox to a cell (right-click → Format Control → Cell link). Then use =COUNTIF(linked range, TRUE) to show how many precautions are "done". This gamifies the safety checklist!

11

Create a chart for your infographic data

From your threats analysis, create a bar chart showing "How Common" each threat type is. This chart can be exported (right-click → Save as Picture) and used in your infographic.

💡 Four key threats to individuals: Invasion of privacy (tracking without consent), Fraud (using your data to steal money/identity), Targeting vulnerable people (children, elderly), Inaccurate data storage (wrong info affecting credit scores, benefits, health records).
